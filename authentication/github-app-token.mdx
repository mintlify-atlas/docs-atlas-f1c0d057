---
title: "GitHub App Token"
description: "Use a GitHub App to generate short-lived tokens for authenticated pushes."
---

GitHub App tokens are the recommended authentication method for production workflows. Compared to long-lived Personal Access Tokens, they offer:

- **Short-lived credentials** — tokens expire after one hour and are regenerated each run, limiting the blast radius of any leak.
- **Fine-grained permissions** — you grant only the repository permissions the app needs, nothing more.
- **Auditability** — activity is attributed to the app installation, not a personal user account, making it easy to audit in the repository's audit log.

The push action accepts any valid token via the `github_token` input — including tokens generated by a GitHub App. You generate the token in an earlier step and pass it through.

<Warning>
The GitHub App must be installed on the target repository and granted **Contents** read and write permission. Without this, the push will be rejected with a `403` error.
</Warning>

## How it works

Use the [`tibdex/github-app-token`](https://github.com/tibdex/github-app-token) action to exchange your app credentials for a short-lived installation token, then pass that token to the push action.

You need three secrets stored in your repository:

| Secret name | What it contains |
|---|---|
| `APP_ID` | The numeric App ID from the GitHub App's settings page |
| `INSTALLATION_ID` | The numeric installation ID for the app on your repository |
| `APP_PRIVATE_KEY` | The PEM-encoded private key generated for the app |

## Complete workflow example

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          ref: ${{ github.head_ref }}
          fetch-depth: 0
          persist-credentials: false

      - name: Generate GitHub App token
        id: generate_token
        uses: tibdex/github-app-token@v1
        with:
          app_id: ${{ secrets.APP_ID }}
          installation_id: ${{ secrets.INSTALLATION_ID }}
          private_key: ${{ secrets.APP_PRIVATE_KEY }}

      - name: Commit files
        run: |
          git config --local user.email "github-actions[bot]@users.noreply.github.com"
          git config --local user.name "github-actions[bot]"
          git commit -a -m "Add changes"

      - name: Push changes
        uses: ad-m/github-push-action@master
        with:
          github_token: ${{ steps.generate_token.outputs.token }}
          branch: ${{ github.ref }}
```

Note the `persist-credentials: false` on the checkout step. This prevents the checkout action from storing the default `GITHUB_TOKEN` as the remote credential, ensuring the app token is used for the push instead.

<Note>
This pattern was first documented in the community in [issue #173](https://github.com/ad-m/github-push-action/issues/173), which covers additional background and edge cases for using GitHub App tokens with this action.
</Note>

## Set up the required secrets

<Steps>
  <Step title="Create or select a GitHub App">
    Go to **Settings** → **Developer settings** → **GitHub Apps** on your GitHub account (or organization) and click **New GitHub App**.

    Under **Repository permissions**, set **Contents** to **Read and write**. Leave all other permissions at their defaults unless your workflow needs them.

    After creating the app, note the **App ID** shown on the app's settings page.
  </Step>
  <Step title="Install the app on the target repository">
    From the app's settings page, click **Install App** and select the repository (or repositories) where the action will run.

    After installation, open the installation URL — it looks like `https://github.com/settings/installations/<installation_id>`. Note the numeric ID at the end of the URL — this is your `INSTALLATION_ID`.
  </Step>
  <Step title="Generate a private key">
    On the app's settings page, scroll to **Private keys** and click **Generate a private key**. Save the downloaded `.pem` file — you will need its contents in the next step.
  </Step>
  <Step title="Add secrets to your repository">
    Navigate to your repository on GitHub, then go to **Settings** → **Secrets and variables** → **Actions** → **New repository secret**.

    Add three secrets:

    - `APP_ID` — the numeric App ID
    - `INSTALLATION_ID` — the numeric installation ID
    - `APP_PRIVATE_KEY` — the full contents of the `.pem` private key file
  </Step>
</Steps>
