GitHub Push Action (`ad-m/github-push-action`) is a GitHub Action that automates pushing local commits back to a GitHub repository from within a workflow. It handles authentication, branch targeting, and advanced push strategies so you can focus on building automation — not wiring up git remotes.

## Why use GitHub Push Action?

When a workflow modifies files — running a linter, generating code, updating a changelog — you need a reliable way to commit and push those changes. GitHub Push Action provides a single, configurable step that handles:

- **Authentication** — works with `GITHUB_TOKEN`, Personal Access Tokens, GitHub App tokens, and SSH Deploy Keys
- **Branch targeting** — push to the same branch, a PR branch, or any branch by name
- **Force push strategies** — standard `--force` or the safer `--force-with-lease`
- **Cross-repository pushes** — push generated content to a separate repository
- **Submodule support** — recursively push submodule changes with a configurable strategy
- **Tag management** — push tags alongside commits or push only tags

## Common use cases

- **Linter auto-fix** — run a formatter, commit the result, push it back
- **Generated file tracking** — archive script outputs in git
- **GitHub Pages publishing** — push a built site to `gh-pages`
- **Repository mirroring** — sync commits to a separate repository
