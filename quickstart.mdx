This guide walks you through adding `ad-m/github-push-action` to a workflow that checks out your repository, makes changes, commits them, and pushes the result back to GitHub.

<Tip>
  Before you begin, make sure your repository has write permissions enabled for the `GITHUB_TOKEN`. See the [permissions setup](/permissions) page for instructions.
</Tip>

## Add the action to your workflow

<Steps>
  <Step title="Check out the repository">
    Add `actions/checkout@v4` as the first step in your job. Set `fetch-depth: 0` so that the full git history is available — without it, the push step may fail when resolving refs.

    ```yaml
    - uses: actions/checkout@v4
      with:
        fetch-depth: 0 # Required to avoid "shallow update not allowed" errors
    ```

    <Note>
      Omitting `fetch-depth: 0` (or leaving it at the default of `1`) creates a shallow clone. A shallow clone does not contain the full ref history that `git push` needs to resolve the destination branch correctly, which causes the push to fail.
    </Note>
  </Step>

  <Step title="Make your changes">
    Run any script, tool, or command that modifies files in the working directory. This is where you would invoke a linter, code generator, or any other automation.

    ```yaml
    - name: Generate or modify files
      run: |
        # Replace this with your actual command, for example:
        npm run build
    ```
  </Step>

  <Step title="Commit the changes">
    Configure the git identity for the commit and create the commit. Using the `github-actions[bot]` identity is the standard convention for machine-generated commits.

    ```yaml
    - name: Commit changes
      run: |
        git config --local user.email "41898282+github-actions[bot]@users.noreply.github.com"
        git config --local user.name "github-actions[bot]"
        git commit -a -m "chore: apply automated changes"
    ```
  </Step>

  <Step title="Push with GitHub Push Action">
    Add the `ad-m/github-push-action@master` step. Pass `secrets.GITHUB_TOKEN` for authentication and `github.ref` to push to the same branch that triggered the workflow.

    ```yaml
    - name: Push changes
      uses: ad-m/github-push-action@master
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        branch: ${{ github.ref }}
    ```
  </Step>
</Steps>

## Complete workflow example

The following is a full, working workflow you can copy into `.github/workflows/push-changes.yml`. It triggers on every push to `main`, runs a placeholder script, commits any resulting changes, and pushes them back.

```yaml
name: Push automated changes

on:
  push:
    branches:
      - main

jobs:
  update:
    runs-on: ubuntu-latest

    permissions:
      contents: write # Required for the push step to write to the repository

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Generate or modify files
        run: |
          # Replace with your actual command
          echo "$(date -u)" > last-run.txt

      - name: Commit changes
        run: |
          git config --local user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git config --local user.name "github-actions[bot]"
          git add -A
          git diff --cached --quiet || git commit -m "chore: apply automated changes"

      - name: Push changes
        uses: ad-m/github-push-action@master
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          branch: ${{ github.ref }}
```
